# Real-time Field Log & Activity Sharing Hub (RtFLASH) 要件定義　基本設計

## システムの目的
STACOSのような工事管理ツールを使うことができないので、
学校の写真台帳はエクセルでみんな作っている。
フリーフォーマットになってしまうので、結構完成図書の品質にばらつきが出るし、
工事後にSEのもとに届くのも遅いため、やり直してもらうことができない。
また、工事進捗を把握するため、別途報告を受ける必要がある。

## ユーザーストーリー
1. 事前に作業をを登録する。作業名といくつかの情報。
1. 作業をする
1. 内容を確認してレビュー指摘を入れたり承認したりする。
1. 進捗状況を確認する。
1. 修正する
1. 完成図書を印刷する

## 機能一覧　実現方式
- [x] 写真を伴う工事情報を登録する機能
  - モデル駆動型アプリのフォームを利用する
- [x] 工事予定の登録機能
  - モデル駆動型アプリのフォームを利用する
- [x] 未登録情報の表示、カウント、予実確認
  - ダッシュボードフィルタ、グラフ機能、行数のカウント（左下）
- [x] 登録データの承認、レビュー、コメント登録等
  - モデル駆動型アプリのフォームを利用する
- [x] デフォルト値登録
  - 状態だけで良さそう
- [x] 登録してほしい情報の登録機能
  - ちょっと難しいので、管理者にはPowerAppsを頑張って使ってカスタマイズしてもらう。
  - または、カスタム入力欄を使ってもらう。
- [ ] 完成図書の出力機能
